# 第二部分：核心机制

## 第五章：Sandbox 与执行环境

### 5.1 为什么 Agent 需要执行环境抽象
LLM 只是决策器，真正动作往往发生在命令执行层。执行环境抽象解决三类问题：
- 安全隔离：限制危险命令影响面。
- 部署弹性：local / docker / ssh 等后端可切换。
- 可重复性：同类任务在不同机器行为一致。

### 5.2 Hermes 的实现策略
项目在 `tools/environments/` 下维护多后端适配器，核心模式是：
- 上层工具只描述“要执行什么”。
- 下层环境适配器决定“在哪里执行、如何执行、如何回收结果”。

### 5.3 与工具调用的结合点
`model_tools.handle_function_call` 对 `execute_code` 有专门分支，按启用工具集传递 sandbox 能力边界，避免子上下文越权。

### 5.4 安全与效率平衡
- 本地模式：调试快，但风险暴露高。
- 容器/远程模式：隔离更好，但带来启动与通信开销。

实务建议：
- 开发阶段 local + 最小权限。
- 生产阶段容器化 + 操作审计 + 只读优先。

### 5.5 fork 实践
在 fork 中新增代码执行能力时，不要直接在工具里写死命令执行逻辑，优先复用已有环境抽象与权限路径。
