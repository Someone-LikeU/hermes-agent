# 第二部分：核心机制

## 第三章：MCP 机制详解

### 3.1 MCP 在 Hermes 中的定位
MCP（Model Context Protocol）在 Hermes 中主要承担“外部资源/工具能力桥接层”角色：
- 把外部系统能力以统一协议接入 Agent。
- 减少核心仓库直接耦合第三方 SDK/数据源。
- 让工具与上下文来源具备跨平台、跨运行模式的复用能力。

### 3.2 与本地工具体系的关系
MCP 并不是替代 `tools/`，而是形成双轨：
1. 内置工具：由 `tools/registry.py` 注册，受 `toolsets.py` 控制暴露。
2. MCP 资源/工具：通过协议桥接提供外部能力。

在运行时，模型看到的是“统一函数调用接口”，但后端实现可以来自本地工具或 MCP provider。

### 3.3 MCP 的工程价值
- **隔离性**：外部变更尽量收敛在 MCP 适配层。
- **扩展性**：新增外部能力通常不必改动 AIAgent 主循环。
- **治理性**：可通过工具白名单、hook、权限策略统一治理。

### 3.4 实际接入建议（fork 视角）
1. 高频/通用能力优先做成协议化 MCP 接口，而不是散落在业务脚本。
2. 对外部资源调用加超时、重试、熔断，避免拖垮主回路。
3. 所有 MCP 输出保持结构化，便于二次处理与审计。
