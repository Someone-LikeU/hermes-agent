# 第二部分：核心机制

## 第二章：Skills 体系

### 3.1 Skills 的系统角色
Skills 是 Hermes 的“可演进 Prompt 资产层”。它把最佳实践从代码中抽离为可管理的知识单元。

### 3.2 双仓设计
- `skills/`：默认内置技能，轻量、通用。
- `optional-skills/`：重依赖或垂类技能，需要显式安装。

这种分层避免默认安装过重，同时保留官方知识资产扩展能力。

### 3.3 运行时接入路径
典型流程：
1. 用户通过 slash command / 配置启用技能。
2. 技能内容在运行时注入消息上下文（通常作为用户侧增强输入）。
3. 模型在同一主循环中消费技能约束与步骤。

### 3.4 与 Curator 的关系
Curator 负责生命周期治理：
- 使用统计（use/view/patch）
- stale/archived 状态迁移
- 备份与回滚

这让 Skills 不只是“静态 prompt 文件”，而是可持续维护的知识系统。

### 3.5 fork 实践建议
- 高频业务流程优先 Skill 化，而不是把提示词硬编码进核心模块。
- 把组织 SOP 写成 Skill，配合工具调用形成“半自动标准作业流”。
- 为 Skills 建立版本标记与变更日志，便于回滚。
