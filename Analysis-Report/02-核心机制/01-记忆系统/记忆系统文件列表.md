# 记忆系统文件列表

> 本文档根据源码检索整理，检索时已排除 `Analysis-Report/` 目录本身。

Hermes-Agent 的记忆系统分为两层：

1. 内置文件型记忆：通过 `memory` 工具维护 `$HERMES_HOME/memories/MEMORY.md` 和 `$HERMES_HOME/memories/USER.md`。
2. 外部记忆提供商：通过 `MemoryProvider` / `MemoryManager` 接入 Honcho、Mem0、Supermemory、Hindsight 等外部或本地后端。

## 一、核心运行时文件

| 文件 | 作用 |
| --- | --- |
| `tools/memory_tool.py` | 内置 `memory` 工具实现，负责读写 `MEMORY.md` / `USER.md`，包含条目增删改、字符限制、文件锁、原子写入、注入风险扫描和工具 schema 注册。 |
| `run_agent.py` | `AIAgent` 中的记忆主接入点：初始化 `MemoryStore`，加载外部 memory provider，把记忆注入 system prompt，路由 `memory` 工具调用，桥接内置记忆写入到外部 provider，并在每轮完成后同步外部记忆。 |
| `agent/memory_provider.py` | 外部记忆后端抽象基类，定义 `initialize`、`prefetch`、`sync_turn`、`get_tool_schemas`、`handle_tool_call`、`shutdown`、`on_session_end`、`on_memory_write`、`on_delegation` 等生命周期接口。 |
| `agent/memory_manager.py` | 外部记忆 provider 的统一编排器。负责 provider 注册、工具 schema 汇总、prefetch、sync、tool routing、session hook、delegation hook、`memory-context` fencing 与上下文清洗。 |
| `model_tools.py` | 把 `memory` 标记为 agent-loop tool，由 `run_agent.py` 拦截执行，而不是走普通工具分发路径。 |
| `toolsets.py` | 将 `memory` 放入核心工具集合 `_HERMES_CORE_TOOLS`，并定义独立的 `memory` toolset。Honcho 已从独立 toolset 改为 memory provider 插件注入。 |

## 二、配置、CLI 和诊断入口

| 文件 | 作用 |
| --- | --- |
| `hermes_cli/config.py` | 默认记忆配置来源：`memory_enabled`、`user_profile_enabled`、`memory_char_limit`、`user_char_limit`、`memory.provider`。 |
| `hermes_cli/main.py` | `hermes memory` 命令入口，包含 `setup`、`status`、`off`、`reset`；`reset` 会删除内置 `MEMORY.md` / `USER.md`。 |
| `hermes_cli/memory_setup.py` | 外部 memory provider 的交互式 setup/status 向导，发现 provider、安装依赖、收集配置、写入 `config.yaml` / `.env` / provider 原生配置文件。 |
| `hermes_cli/plugins_cmd.py` | `hermes plugins` 中的 provider 选择逻辑，读取/保存 `memory.provider`，并列出可用 memory providers。 |
| `hermes_cli/doctor.py` | `hermes doctor` 中检查当前 memory provider：内置、Honcho、Mem0 及其他通用 provider。 |
| `hermes_cli/dump.py` | `hermes dump` 输出当前激活的 memory provider。 |
| `hermes_cli/profiles.py` | profile 导入/导出与克隆时处理 `memories/MEMORY.md` 和 `memories/USER.md`。 |

## 五、会话边界、隔离与其他调用面

| 文件 | 作用 |
| --- | --- |
| `cli.py` | CLI 退出、新建 session、resume、branch 时调用 `shutdown_memory_provider`、`commit_memory_session` 或 `on_session_switch`，保证 provider 写入目标 session 正确。 |
| `gateway/run.py` | gateway 中清理 agent、session resume、缓存 agent shutdown 时处理 memory provider，避免 provider 持有旧 session_id。 |
| `tui_gateway/server.py` | TUI session 关闭/压缩前通过 `commit_memory_session` 触发 provider session-end 提取。 |
| `tools/delegate_tool.py` | 子代理默认 `skip_memory=True`，但父代理会通过 `on_delegation` 把 delegation 任务和结果通知自己的 memory provider。 |
| `hermes_state.py` | 在读取历史消息时使用 `sanitize_context` 去除注入的 `<memory-context>` 块，避免记忆上下文泄漏进 session 搜索/历史。 |
| `batch_runner.py` | batch 任务创建 agent 时使用 `skip_memory=True`，避免批处理污染持久记忆。 |
| `cron/scheduler.py` | cron agent 使用 `skip_memory=True`，避免定时任务 system prompt 污染用户记忆。 |
| `agent/curator.py` | 后台 review/curator 场景使用 `skip_memory=True`，避免维护任务写入主记忆。 |
| `gateway/platforms/feishu_comment.py` | 部分平台辅助 agent 使用 `skip_memory=True`。 |
