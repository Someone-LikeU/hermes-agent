# 运行时 Prompt 构建涉及文件清单

## 核心编排

- `run_agent.py`
  - `AIAgent._build_system_prompt()`：稳定 system prompt 的总装入口。
  - `AIAgent.run_conversation()`：复用/存储 system prompt，构造 API 消息，注入 ephemeral prompt、prefill、memory prefetch、plugin context。
  - `_compress_context()`：上下文压缩后重建 system prompt。
  - `_build_api_kwargs()`：按 API mode 分发到 transport。

- `agent/prompt_builder.py`
  - 定义默认身份、memory/session/skills/kanban guidance、tool-use enforcement、平台提示、环境提示。
  - `build_skills_system_prompt()`：生成 skills 索引。
  - `build_context_files_prompt()`：加载 `.hermes.md/HERMES.md/AGENTS.md/CLAUDE.md/.cursorrules`。
  - `load_soul_md()`：加载 `~/.hermes/SOUL.md` 作为身份层。

- `hermes_state.py`
  - `sessions.system_prompt` 字段保存稳定 system prompt 快照。
  - `update_system_prompt()` 在首轮构建或压缩重建后写入快照。

## API 消息适配

- `agent/transports/chat_completions.py`
  - Chat Completions 请求组装。
  - GPT-5/Codex 模型把首条 `system` role 改成 `developer` role。

- `agent/transports/codex.py`
  - Codex Responses API 请求组装。
  - 从首条 system message 提取 `instructions`。

- `agent/transports/anthropic.py`
  - Anthropic Messages 请求组装，委托 Anthropic adapter 拆分 system 与 messages。

- `agent/transports/bedrock.py`
  - Bedrock Converse 请求组装。

## 记忆注入

- `tools/memory_tool.py`
  - `MemoryStore.load_from_disk()` 读取 `MEMORY.md`、`USER.md` 并冻结 system prompt snapshot。
  - `format_for_system_prompt()` 返回冻结快照，保证 mid-session 记忆写入不改变 system prompt。

- `agent/memory_manager.py`
  - `build_system_prompt()` 收集外部 memory provider 的 system prompt block。
  - `prefetch_all()` 当前轮预取记忆上下文。
  - `build_memory_context_block()` 把预取结果包进 `<memory-context>` 注入 user message。

- `agent/memory_provider.py`
  - 外部 memory provider 接口，定义 `system_prompt_block()`、`prefetch()` 等 prompt 相关扩展点。

## Skills 注入

- `agent/skill_commands.py`
  - `build_skill_invocation_message()`：slash skill 调用时把完整 skill 内容构造成用户消息。
  - `build_preloaded_skills_prompt()`：CLI/TUI 启动预加载 skill，并拼入 ephemeral system prompt。
  - `_build_skill_message()`：拼接 skill 正文、skill 目录、配置值、setup note、supporting files。

- `agent/skill_preprocessing.py`
  - skill 内容模板变量替换与 inline shell 展开。

- `agent/skill_utils.py`
  - 解析 skill frontmatter、平台过滤、禁用列表、外部 skill 目录。

- `tools/skills_tool.py`
  - `skill_view` 加载 skill 正文；`SKILLS_DIR` 定位本地 skills。

## 项目上下文与渐进提示

- `agent/subdirectory_hints.py`
  - 工具访问新目录时，延迟读取该目录附近的 `AGENTS.md/CLAUDE.md/.cursorrules`，追加到 tool result。

- `hermes_constants.py`
  - `get_hermes_home()`、`get_skills_dir()` 等 profile-aware 路径，影响 SOUL、memory、skills 的实际来源。

## CLI / TUI / Gateway 入口

- `cli.py`
  - 读取 `HERMES_EPHEMERAL_SYSTEM_PROMPT`、`agent.system_prompt`、`prefill_messages_file`。
  - 创建 `AIAgent` 时传入 `ephemeral_system_prompt`、`prefill_messages`。
  - 处理 `--skills` 预加载、worktree note、slash skill、`/reload-skills` one-shot note。

- `tui_gateway/server.py`
  - TUI 创建 `AIAgent`，读取 `agent.system_prompt`，处理启动 skill 预加载，传入 ephemeral prompt。

- `gateway/run.py`
  - Gateway 读取全局 ephemeral prompt 与 prefill。
  - 组合平台 context、channel prompt、用户配置 prompt 后传给 `AIAgent`。
  - 处理 messaging 平台的 slash skill 与 skills reload note。

- `gateway/platforms/api_server.py`
  - OpenAI-compatible API Server 把请求中的 `system` messages 合并为 `ephemeral_system_prompt`，再调用 agent。

- `gateway/platform_registry.py`
  - 插件平台可提供 `platform_hint`，作为 platform-specific system prompt block。

- `gateway/session_context.py`
  - 提供平台/session 环境变量，影响 skills 索引缓存 key 与平台过滤。

## 配置与工具集合

- `hermes_cli/config.py`
  - `DEFAULT_CONFIG` 中定义 `agent.system_prompt`、`agent.prefill_messages_file`、`agent.tool_use_enforcement`、skills 配置等默认值。

- `model_tools.py`
  - 工具发现与有效工具名集合来源；`valid_tool_names` 决定哪些 guidance 和 skills 索引被注入。

- `toolsets.py`
  - 工具集定义；当前启用 toolsets 影响可用工具、skills 条件过滤与最终 tool schemas。

- `tools/registry.py`
  - 工具 schema 注册中心；tool schemas 作为 API 的 `tools` 参数传递，不拼进文本 system prompt。

## 插件与扩展上下文

- `hermes_cli/plugins.py`
  - `pre_llm_call` hook 可返回上下文；该上下文只注入当前 user message，不进入 system prompt。

- `plugins/memory/*`
  - 外部 memory provider 实现可能提供 system prompt block 与 per-turn recall。

- `plugins/platforms/*`
  - 平台插件可能通过 registry 提供 platform hint。

## 行为覆盖测试

- `tests/agent/test_prompt_builder.py`
  - 覆盖 context file 加载优先级、注入扫描、skills 索引、环境提示、tool-use guidance 等行为。
